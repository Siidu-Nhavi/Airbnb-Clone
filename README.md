## ğŸ” User Authentication & Authorization (Session-Based)

Wanderlust implements a **secure session-based authentication system** using **Passport.js** and **MongoDB** to manage user login state and access control.

---

### âœ¨ Features Implemented
- User signup with **username, email, and password**
- Secure login using **Passport.js (Local Strategy)**
- Password hashing and salting with **passport-local-mongoose**
- Session management using **express-session**
- Persistent session storage using **connect-mongo**
- Flash messages for authentication success and errors
- Automatic redirect after successful login
- Fully implemented **User model** and **authentication routes**
- Protected routes using authorization middleware

---
 ğŸ”„ How Session-Based Authentication Works

### 1ï¸âƒ£ User Registration
- Users sign up by providing a username, email, and password.
- Passwords are **hashed and salted** automatically using `passport-local-mongoose`.
- User data is stored securely in MongoDB.

---

### 2ï¸âƒ£ User Login (Authentication)
- User submits login credentials.
- Passport.js:
  - Fetches the user from MongoDB.
  - Verifies the password against the stored hash.
- On success, Passport authenticates the user and initiates a session.

---

### 3ï¸âƒ£ Session Creation
- After successful authentication:
  - A **session ID** is generated by the server.
  - The session ID is stored in the browser as a **cookie**.
- No sensitive data is stored on the client.

---

### 4ï¸âƒ£ Session Storage (MongoDB)
- Sessions are stored in MongoDB using **connect-mongo**.
- This enables:
  - Persistent login across server restarts.
  - Scalability for production deployment.

---

### 5ï¸âƒ£ Session Persistence
- On every request:
  - The browser sends the session cookie automatically.
  - Express-session retrieves session data from MongoDB.
  - Passport restores the logged-in user using:
    - `serializeUser()`
    - `deserializeUser()`
- The authenticated user is accessible via:
```js
req.user

6ï¸âƒ£ Authorization (Protected Routes)

req.isAuthenticated()

Authenticated users can:
-Create listings.

-Unauthorized users are redirected to the login page with a flash message.

7ï¸âƒ£ Flash Messages
Flash messages provide feedback for:
-Successful login or signup.

-Messages persist for one request only, improving user experience.

8ï¸âƒ£ Logout
When a user logs out:
-The session is destroyed.
-The session cookie is cleared.
-The user is redirected safely.

âœ… Why Session-Based Authentication?
-Secure server-side session handling
-Prevents exposure of sensitive data on the client
-Ideal for server-rendered applications (EJS)
-Easily extendable to role-based authorization (Admin/User)
